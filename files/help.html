Nápověda

Co aplikace dělá

Aplikace slouží k učení pomocí kartiček (otázka → odpověď). Témata jsou složky ve složce data, každé téma obsahuje jedno nebo více cvičení (CSV soubory) a volitelně složku Pic s obrázky s vysvětlením.

První stránka — témata

  • Zobrazuje všechna témata (názvy složek, podtržítka _ se zobrazují jako mezery).
  • Klepnutím na téma se rozbalí seznam jeho cvičení s procentem naučení, např. Axa_sklonovani_clen_urcity_1 (70%). Klepnutím na cvičení se otevře stránka učení.
  • Stats — zobrazí přehled učení: pokrok všech cvičení, počet kol, datum posledního učení a historii posledních událostí.
  • Pin — zapne „režim připínání“ (tlačítko zčervená). V tomto režimu klepnutím na téma téma připnete nahoru (📌), dalším klepnutím odepnete. Opětovným stiskem tlačítka Pin režim ukončíte.
  • Help — otevře tuto nápovědu.

Stránka cvičení — jak se učit

  1. Vidíte první sloupec (otázku). Druhý sloupec (odpověď) je skrytý.
  2. Klepnutím na skrytou odpověď ji zobrazíte. Pokud jste odpověď věděli, nechte ji zobrazenou. Pokud ne, klepněte znovu a odpověď se opět skryje.
  3. Až projdete celý seznam, stiskněte dole tlačítko Next round. Řádky, které jste nechali zobrazené (uměli jste je), zmizí a v dalším kole zůstanou jen ty, které neumíte.
  4. Opakujte, dokud nezbude nic — cvičení je na 100 %.

Pokrok v procentech = naučené řádky ÷ všechny řádky.

Poznámky (třetí sloupec)

Pokud má řádek poznámku, třetí úzký sloupec má světle zelené pozadí. Klepnutím na něj se poznámka rozbalí přes celý řádek, dalším klepnutím se opět sbalí.

Tlačítka na stránce cvičení

  • Back — návrat na seznam témat.
  • Reset — smaže pokrok tohoto cvičení, začnete znovu od 0 % (aplikace se předtím zeptá).
  • Pin — připne/odepne toto téma na první stránce.
  • Pic — zobrazí obrázky s vysvětlením ze složky Pic daného tématu. Při jednom obrázku je k dispozici tlačítko Exit, při více obrázcích také Back a Forward pro listování.

Ukládání stavu

Aplikace si pamatuje vše (pokrok, rozdělané kolo, připnutá témata, poslední otevřenou stránku, historii) v úložišti prohlížeče (localStorage). Po opětovném otevření pokračujete přesně tam, kde jste skončili.

Pozor: úložiště je vázané na konkrétní prohlížeč a konkrétní webovou adresu. Pokrok na adrese u Vercelu a na druhém hostingu jsou tedy oddělené. Smazáním dat prohlížeče se pokrok smaže.

Synchronizace mezi zařízeními

Pokrok se může automaticky sdílet mezi vašimi zařízeními (telefon, počítač) přes váš účet GitHub — ukládá se do malého soukromého souboru (gist). Zapnutí na každém zařízení:

  1. Otevřete Stats → sekce Synchronizace.
  2. Klepněte na odkaz Vytvořte token — otevře se GitHub s předvyplněným formulářem, dole klepněte na zelené Generate token a token zkopírujte (začíná ghp_).
  3. Token vložte do políčka a klepněte na Připojit.

Od té chvíle se pokrok ukládá na GitHub automaticky (chvíli po každé změně) a při otevření aplikace se stáhne. Když se učíte na více zařízeních, u každého cvičení platí novější stav. Token je uložený jen v prohlížeči daného zařízení; tlačítkem Odpojit synchronizaci na zařízení vypnete. Stejný token můžete použít na všech svých zařízeních.

Formát dat

Struktura složek (vedle learn.html):

data/
  Nazev_tematu/            ← podtržítka místo mezer
    Nazev_tematu_1.csv
    Nazev_tematu_2.csv
    Pic/
      obrazek1.png

CSV soubor má tři sloupce: otázka, odpověď, poznámka (poznámka může být prázdná). Oddělovačem může být čárka nebo středník — aplikace ho pozná automaticky. Text s čárkami, uvozovkami nebo více řádky musí být v uvozovkách "…".

Přidání nebo změna témat

  1. Přidejte/zmeňte složky a CSV soubory ve složce data.
  2. Vygenerujte nový seznam témat příkazem: 
    python3 files/build_manifest.py
  3. Nahrajte změny na GitHub — web se sám znovu nasadí.

Drobné opravy v CSV lze dělat přímo na github.com: otevřete soubor, klepněte na tužku, upravte a uložte (Commit). Web se do minuty sám aktualizuje.

Spuštění

Aplikace musí běžet přes web server (Vercel, GitHub Pages, nebo lokálně např. python3 -m http.server ve složce aplikace). Při otevření souboru learn.html přímo z disku (file://) prohlížeč nedovolí načíst data.
